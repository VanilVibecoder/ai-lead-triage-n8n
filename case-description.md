# AI Lead Triage — тренировочный портфолио-кейс

## Статус проекта

Это тренировочный портфолио-кейс, не клиентское внедрение.

Кейс создан на основе реального бизнес-паттерна:
входящая заявка → валидация → проверка дублей → AI-квалификация → Google Sheets → Telegram-уведомление менеджеру.

## Проблема

У онлайн-школ, экспертов и сервисных бизнесов заявки часто приходят из разных каналов: сайт, форма, Telegram, реклама.

Менеджеры вручную читают сообщения, определяют срочность, копируют данные в таблицу и решают, что делать дальше.

Из-за этого горячие заявки могут обрабатываться поздно, дубли путаются с новыми лидами, а спам отвлекает менеджера.

## Решение

Workflow на n8n принимает заявку через защищённый webhook, проверяет secret header, валидирует phone/email/message, нормализует контакты, проверяет дубли, классифицирует заявку через AI, считает lead_score и SLA, сохраняет данные в Google Sheets и отправляет менеджеру Telegram-карточку.

## Стек

- n8n
- Mistral AI
- Google Sheets
- Telegram Bot
- JavaScript Code Node
- Webhook
- Postman

## Что реализовано

- Webhook security через secret header
- Валидация входных данных
- Нормализация телефона и email
- Duplicate check по lead_key
- AI-классификация заявки
- Priority: hot / warm / cold / unknown
- Lead score 0–100
- SLA и дедлайн ответа
- Запись в Google Sheets
- Telegram-уведомление менеджеру
- Spam routing
- Fallback при невалидном AI JSON
- Error Workflow при технической ошибке

## Тестовые сценарии

1. Новая горячая заявка
2. Повторная горячая заявка
3. Сохранение в Google Sheets
4. Unauthorized request 401
5. Error Workflow alert

## Результат

Менеджер получает не сырой текст заявки, а готовую карточку: кто написал, что хочет, насколько срочно, какой score, когда нужно ответить и что делать дальше.
